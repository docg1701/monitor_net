# Story 3.3: v1.0 Final Verification

## Status
Ready for Review

## Story
**As a** Product Owner,
**I want** to verify the v1.0 Release artifacts on all platforms,
**so that** I can confidently publish the official v1.0 release to users.

## Acceptance Criteria
1. A `v1.0.0` tag triggers the full release workflow.
2. Release artifacts for Linux, Windows, macOS, and Android are successfully generated and attached to the GitHub Release.
3. The version number is consistently `1.0.0` in `package.json`, `src-tauri/tauri.conf.json`, and `netmonitor/android/app/build.gradle`.
4. The Desktop application installs and runs on Linux (local verification). Windows and macOS builds are verified via CI success logs only.
5. The Android APK installs and runs on a device/emulator.
6. Network monitoring (including the Ping fix) works on all tested platforms (Linux & Android).

## Dev Notes
**Context:**
This is the final verification story for Epic 3. We are validating that the CI/CD pipelines built in 3.1 and 3.2 produce working v1.0 artifacts.

**Environmental Constraints:**
- User can ONLY manually verify on **Linux** (Desktop) and **Android** (Emulator).
- Windows and macOS verification relies entirely on the GitHub Actions "Success" status.

**Previous Story Insights:**
- Story 3.2 fixed Android Ping using `CapacitorHttp`.
- `MonitorService` (src/app/services/monitor.service.ts) has logic to switch between Tauri (Desktop) and CapacitorHttp (Mobile/Web). This **MUST** be verified on Desktop to ensure the fallback logic doesn't regress Desktop functionality.

**File Locations:**
- **Versioning**:
    - `netmonitor/package.json` [Source: architecture/unified-project-structure.md]
    - `netmonitor/src-tauri/tauri.conf.json`
    - `netmonitor/android/app/build.gradle`
- **Release Workflow**: `.github/workflows/release.yml`

**Technical Details:**
- **Versioning Strategy**: All three files must match "1.0.0". Android `versionCode` must be an integer (e.g., 2 or 10000).
- **Signing**:
    - Desktop: Self-signed/Unsigned (Standard for MVP).
    - Android: Signed via CI secrets (verified in 3.2).
- **Verification**:
    - **Linux**: Local execution is primary.
    - **Android**: Emulator/Device execution is primary.
    - **Win/Mac**: CI success is sufficient for MVP, unless a Windows/Mac machine is available for manual test.

**Project Structure Alignment:**
- Files are in standard locations per `unified-project-structure.md`.
